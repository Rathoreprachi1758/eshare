Frequently Asked Questions

The eShare.ai comprehensive file encryption architecture is designed to offer up to three levels of file encryption to ensure that your sensitive data is always protected and secure.

In-transit file encryption is active by default and designed with the use of HTTPS connections and the latest TLS protocol. As a user of the eShare.ai comprehensive file encryption application, you can encrypt your files server-wide and/or end-to-end. In fact, in-transit encryption is mandatory to comply with the General Data Protection Regulation (GDPR).

Encryption at rest refers to encrypting files from the eShare.ai application server before saving them on storage. In this setup, eShare.ai uses a single master critical encryption method, allowing only the administrator holding the key to decrypt the files, ensuring high protection against data breaches, even in a hardware breach. The keys can be stored in a hardware security module (HSM) for added security.

eShare.ai provides an End-to-End (E2EE) file encryption plugin strongly recommended for highly valuable and sensitive data. This plugin enables users and authorized guest users to share fully encrypted files across all modern browsers and via email. A JavaScript plugin encrypts files on upload with public keys provided by the server. File decryption is enabled with the use of a private key in the web interface.

As soon as your eShare.ai administrator enables the app, all your eShare.ai data files are automatically encrypted. File Encryption is server-wide. So, once enabled, keeping your files unencrypted is not possible. You can use your eShare.ai login credentials as the password for your unique private encryption key. Certain resources, including but not limited to file names, image thumbnails, existing files in the trash bin, file previews, the search index from the full-text app, third-party app data, and existing files in Versions, are not encrypted.
